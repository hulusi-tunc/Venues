# Technical Specifications : Venues

> Non-functional requirements, standards, and conventions.

## 1. Performance

| Requirement | Target | Measurement | Status |
|---|---|---|---|
| API response time | < 300 ms (P95) | Application monitoring | Target (no APM in place yet) |
| Page load (LCP) | < 2.5 s | Lighthouse | Measurable |
| Time to Interactive (TTI) | < 3.5 s | Lighthouse | Measurable |
| Search results response | < 2 s | Application monitoring | Target (no APM in place yet) |
| WebSocket latency | < 100 ms | Network profiling | Target |
| Image optimization | WebP, lazy loading, responsive sizes | Build report | Implemented |
| Bundle size (frontend) | < 500 KB initial JS | Build report | Measurable |

## 2. Scalability

| Requirement | Target | Status |
|---|---|---|
| Concurrent users | 500 (launch), 2000 (post-launch growth) | Target (not load-tested) |
| Venues in database | 1500+ | Target |
| Monthly page views | 50K (launch) | Target |
| File storage | AWS S3 | Configured |
| Database scaling | PostgreSQL replica set | Deferred post-launch (single instance at go-live) |

## 3. Availability

| Requirement | Target | Status |
|---|---|---|
| Production uptime | 99.5% | Target (no SLA contractualized) |
| RTO (Recovery Time Objective) | < 4 hours | Target (not yet validated by drill) |
| RPO (Recovery Point Objective) | < 24 hours | Effective (daily backup in place) |
| Backup frequency | Daily (PostgreSQL dump) | Configured |
| Backup retention | 30 days | Configured |

## 4. Security

| Requirement | Description |
|---|---|
| Authentication | JWT tokens via @nestjs/passport + passport-jwt. Token expiry configured via environment. |
| Password storage | bcrypt hashing |
| SSO | Google OAuth, LinkedIn OAuth |
| API security | Helmet headers, CORS whitelist, rate limiting |
| Admin forgot-password rate limit | 10 attempts per hour |
| Admin login rate limit | Implemented via @nestjs/throttler |
| OTP expiry | 15 minutes |
| File upload validation | Type whitelist (images, docs, audio, video) |
| In-transit encryption | HTTPS (TLS 1.2+) on all environments |
| RGPD | Account deletion, data export capability |
| Session management | Stateless JWT authentication via NestJS Passport module. No server-side sessions. |
| Input validation | Zod + Yup (frontend), class-validator + class-transformer (backend) |

## 6. Development Conventions

### API

- Style: REST, JSON
- URL: `/api/v1.0/` prefix
- Authentication: JWT Bearer tokens
- Errors: standardized JSON format
- Documentation: auto-generated via @nestjs/swagger (OpenAPI 3.0)

### Code

| Area | Convention |
|---|---|
| Language (frontend) | TypeScript (Next.js) |
| Language (admin) | TypeScript (Vite + React) |
| Language (backend) | TypeScript (NestJS) |
| Backend linter | ESLint (@typescript-eslint) |
| Frontend formatter | Prettier |
| Testing (frontend) | Jest + React Testing Library |
| Testing (admin) | Vitest |
| Commits | Conventional Commits (`feat:`, `fix:`, `chore:`, etc.) |
| Branches | main, dev, staging, feature/*, fix/* |
| Code review | Required before merge to dev |

### Database

- Engine: PostgreSQL 16 (TypeORM 0.3.25)
- Naming: camelCase for entity fields, snake_case for table names (TypeORM convention)
- Indexes: on all foreign keys (auto-generated by TypeORM) and frequently searched fields. Elasticsearch for full-text venue search.
- Soft delete: for business entities (venues, bookings, users)

## 7. Standards and Protocols

| Standard | Scope | Requirements |
|---|---|---|
| RGPD / GDPR | Data protection | Consent management, right to deletion, data export, cookie consent banner, privacy policy |
| PCI-DSS | Payment security | Delegated to Stripe (PCI Level 1 certified). No card data stored or processed on platform servers. |
| OWASP Top 10 | Application security | Input validation, parameterized queries, CSRF protection, security headers (Helmet), dependency audit, rate limiting on auth endpoints |
| REST conventions | API design | JSON request/response format, standard HTTP status codes (200, 201, 400, 401, 403, 404, 500), versioned endpoints (`/api/v1.0/`), consistent error response structure |
